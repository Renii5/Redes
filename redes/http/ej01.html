Preguntas sobre HTTP

1) ¿Sobre qué protocolo binario está montado el protocolo HTTP?
HTTP no está montado sobre un protocolo binario; es un protocolo basado en texto que generalmente se transporta sobre TCP (Transmission Control Protocol).
2) ¿Cuáles son los clientes HTTP y los servidores HTTP más utilizados?
Clientes HTTP: Google Chrome, Mozilla Firefox, Microsoft Edge, Safari, cURL, wget, Axios, Requests.
Servidores HTTP: Apache, Nginx, Microsoft IIS, LiteSpeed, Caddy.
3) ¿Qué verbos admite un comando en el requerimiento HTTP?
GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS, CONNECT, TRACE.
4) ¿Qué contenido lleva el body de un requerimiento HTTP?
Datos de formularios, JSON, archivos binarios, texto plano, XML, entre otros formatos.
5) ¿Qué diferencia existe entre un URL y un URI?
Todos los URLs son URIs, pero no todos los URIs son URLs. Un URL es un tipo específico de URI que proporciona la ubicación de un recurso.
6) ¿Cómo se almacena información relacionada con las respuestas HTTP?
Cookies, caché, session storage, local storage, headers HTTP.
7) ¿Qué significa Virtual Hosting?
Virtual Hosting permite a un servidor web alojar múltiples dominios en un solo servidor físico o dirección IP.
8) ¿Por qué el HTTP se considera STATELESS?
Porque cada solicitud HTTP es independiente y no guarda información sobre solicitudes previas.
9) ¿Qué nueva versión de HTTP se está usando para mejorar la velocidad de la web?
HTTP/3, que utiliza el protocolo QUIC basado en UDP.
10) ¿Cómo va a mejorar el comportamiento de una aplicación web en ambientes ruidosos o de débil conexión?
HTTP/3 y QUIC mejoran la velocidad y resistencia en conexiones inestables.
